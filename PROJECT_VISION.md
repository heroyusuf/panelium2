# PROJECT VISION

## Project Name
Automated Pannellum Virtual Tour

---

## Project Goal

Develop a modern, modular, and extensible panoramic virtual tour platform.

The project must evolve from the existing codebase.

Never create a new project.

Always improve and extend the current implementation.

---

## Main Objectives

- Panorama Viewer
- Visual Tour Editor
- Scene Management
- Hotspot System
- Popup System
- Skin System
- Asset Manager
- Theme Engine
- Export System
- Plugin System
- Localization
- Project Save / Load

---

## UI Policy

The existing HTML editor interface is protected.

Do NOT redesign it.

Do NOT replace it.

Do NOT move panels.

Do NOT change toolbar layout.

Only add new functionality.

---

## Reference Folders

The following folders are architectural references:

- 3d vista ornek
- skin ornek
- ornek 1

- jpeg

These folders exist only for analysis.

Never copy code.

Never copy assets.

Never recreate the same UI.

Extract engineering concepts only.

---

## Architecture Philosophy

Prefer:

- Modular
- Reusable
- Plugin-ready
- Clean Architecture
- SOLID
- DRY
- Maintainable
- High Performance

---

## MVP Scope

The first working version must include:

- Panorama Viewer
- Scene Manager
- Hotspot Editor
- Asset Manager
- JSON Project Save / Load
- Popup Engine
- Skin Engine
- Export Package

Everything else can be added later.

---

## Long-Term Vision

The software should eventually become comparable in capability to professional panoramic tour software while remaining an original implementation.

Future modules may include:

- Plugin Marketplace
- VR Support
- WebXR
- Point Cloud
- Gaussian Splatting
- AI Assistant
- Cloud Sync
- Multi-user Editing

---

## Absolute Rules

Never rewrite the project.

Never restart from scratch.

Always continue from the current project state.

Reuse existing code whenever possible.

Every new feature must integrate into the existing architecture.
